发推送领赏：
CDKEY = 用户ID xor Encrypt(发推送日期)
活动签到：
CDKEY = Encrypt（活动类型*1e8+活动日期）
金币存储：
每个用户有一个记录在文件里的01状态串str
合法范围内的日期有4*12*31=1488个。
每32位为一个块，块的后31位为有效信息，首位为奇偶校验位
1~48块：第一类活动签到信息
49~96块：第二类活动签到信息
...
385~432块：第九类活动签到信息

433~480块：推送领赏信息

481~528块：预留分享项目1领赏信息
...
673~720块：预留分享项目5领赏信息

721~800块：竞答题目是否已答
801~880块：竞答题目是否答对

881~1024块：奖品信息及预留空间
